Fábrica de Software

Os serviços prestados para fábrica de software permitem que esta expanda o seu negócio e atenda a um número maior de clientes. A produção de uma fábrica desse tipo é maximizada com o uso do software WCT, o que significa que os recursos, humanos ou ferramentas, alocados em produção, são aproveitados ao máximo pela empresa. Além disso, a customização do WCT e os adaptadores desenvolvidos por demanda permitem uma boa sinergia entre membros de uma equipe, que é alocada na linha de produto da fábrica, e uma boa compatibilidade entre as ferramentas utilizadas no desenvolvimento das aplicações.
WorkCASE Toolkit (WCT)

O software WCT traz para a fábrica de software maior produtividade. Ele pode ser entendido como uma máquina, que recebe como entrada o modelo de um sistema, aplica transformações neste modelo, e gera como saída os dados que são utilizados para compor uma aplicação. Assim, o WCT permite à empresa reduzir os prazos de entrega dos aplicativos de seus clientes e, como consequência, obter maiores lucros.

WCT utiliza a abordagem de desenvolvimento denominada como Agile Model Driven Architecture (AMDA), que traz agilidade na criação de aplicativos com base em modelos de software.
Adaptadores de Processos de Transformação de Modelos

O desenvolvimento de adaptadores para processos de transformação de modelos possibilita à fábrica de software definir os processos de execução das transformações. As transformações recebem como entrada modelos de sistemas e geram como saída artefatos de software. Os processos de transformações possibilitam, portanto, configurar a máquina de transformação que gera automaticamente ou de maneira guiada alguns dos artefatos utilizados em uma aplicação.
Automatização das Linhas de Produçao da Fábrica de Software

Uma linha de produção configura tanto o processo de produção de uma fábrica de software quanto os recursos que são alocados em produção por ela. A ADAPIT provê produtos e serviços para automatizar as linhas de produção. A automatização traz um alto valor agregado para a empresa e uma melhoria considerável na produtividade dos mebros da equipe de desenvolvimento de sistemas.

A ADAPIT fornece o produto WCT, que permite automatizar linhas de produção. Os serviços da ADAPIT permitem customizar o WCT para a empresa cliente. Assim, garantimos que a automatização da linha de produção tem um rápido retorno sobre o capital investido nas soluções fornecidas pela ADAPIT.
Modelo de Sistema

Um modelo de um sistema contém um conjunto de dados que descreve uma aplicação. Fábricas de software tipicamente iniciam pela especificação desses dados em uma etapa chamada de modelagem e projeto de sistemas.

O software WorkCASE Toolkit (WCT) é um sistema para desenvolvimento de software dirigido por modelos (MDD). Ele recebe como entrada o modelo de um sistema em formato XML, XMI ou CMOF, e aplica transformações automáticas ou guiadas por um processo de transformação.
Interoperabilidade em Modelos e Compatibilidade de Ferramentas de Produção de Software

Nas fábricas de software, os dados de uma aplicação podem ser especificados com diversas ferramentas de produção. Porém, para a maioria delas os modelos não são compatíveis, o que ocasiona um problema de troca de informação entre elas (interoperabilidade). Como consequência disso, a fábrica torna-se dependente de determinados fornecedores e pode aproveitar de apenas alguns benefícios desses sistemas.

O WCT pode ser adaptado para suportar diferentes formatos de arquivos usados em diferentes ferramentas de produção de software. Ele possibilita que a fábrica de software opere nos modelos de sistemas utilizando diversas ferramentas de produção de software.
Entrada de Dados para Processamento pela WCT

A entrada para o WCT é um modelo de um sistema. Este modelo contém dados, que representam os requisitos de uma aplicação em alto nível, simples e de fácil entendimento. São exemplos desses dados os formulários de cadastro, as telas de uma aplicação, o modelo do banco de dados, as interações dos usuários com o sistema, dentre outros. Eles são transformados pelo WCT em artefatos de sistemas, que são blocos de construção utilizados para desenvolver uma aplicação.

Portanto, o WCT converte os dados de um sistema especificados rapidamente em um modelo de entrada em outros dados mais complexos.

Adaptadores de Tecnologias

Uma fábrica de software pode utilizar muitas tecnologias na sua linha de produto. A ADAPIT oferece o serviço de customização do WCT para atender as tecnologias utilizadas pela fábrica. A adaptação de tecnologias permite que a linha de produto gere como saída artefatos de software mapeados para as arquiteturas, linguagens de progração, padrões de projeto e demais recursos usados em produção na fábrica de software.
Artefatos de Software

Os artefatos de software são dados que especificam partes de uma aplicação. São resultantes da transformação de um modelo de entrada pelo WCT. Um dado de entrada contido no modelo pode ser uma interface de usuário enquanto o artefato resultante é o código da aplicação para a tecnologia a qual executa a interface de usuário. Os artefatos de software são resultados do processo de transformação da linha de produto da fábrica de software.
Aplicação

A aplicação é um software que pode ser instalado e executado em um determinado computador. Ela é o resultado do trabalho realizado na fábrica de software.

Os produtos e serviços oferecidos pela ADAPIT possibilitam o desenvolvimento de aplicações em prazos menores e com valores mais acessíveis aos seus clientes.

Onde Estão as Pessoas?

O esquema acima apresentou a fábrica, mas onde estão as pessoas que fazem o trabalho acontecer? Elas estão distribuídas em cada uma das atividades e são elas as construtoras dos artefatos. Cada pessoa é responsável por uma característica importante do processo de desenvolvimento como: especialistas que trabalharão com modelagem; outros com análise de requisitos; outros com rastreabilidade e planejamento; outras com adaptações da WCT para atender as necessidades da fábrica; e, por fim, muitas outas programarão as funcionalidades pois a WCT é apenas um software de complemento, não um ambiente completo.

WCT Não é um Framework

A WCT não é um framework. Apesar de o sistema licença server ser disponibilizado junto com um pequeno framework de controle de acesso, este serve apenas para uma configuração padrão disponibilizada pela Adapit para aqueles que não tenham definida uma arquitetura. WCT deve ser configurado para gerar código para o framework utilizado na fábrica de software.

WCT Não é uma Plataforma

WCT não é uma plataforma de execução de aplicativos. Em muitos sistemas de produção você fará o protótipo e testará logo em seguida numa infra-estrutura de execução, sendo ela necessária em todas as implantações do sistema. Esse não é o caso da WCT. Temos um projeto futuro para realizar testes funcionais utilizando o modelo, porém o mesmo não deve ser entendido como um ifra-estrutura de execução da aplicação.

Os objetivos na WCT são apenas o de prototipar a aplicação e gerar o código mais detalhado possível para a sua arquitetura. A infra-estrutura de execução da sua aplicação você é quem decide. Até mesmo o workflow da sua aplicação precisa ser definido por você, personalizando desde onde os botões de uma tela de CRUD vão estar, assim como o local da tela em que resultados serão apresentados ao usuário. O resultado da WCT deve ser código para uma aplicação normal como as desenvolvidas por sua equipe de programação.

WCT Não é uma IDE

A WCT não é uma IDE. Significa que você não vai utilizá-la para escrever códigos Java. Para isso você vai utilizar uma IDE como Eclipse ou NetBeans e ao gerar o código-fonte pela WCT selecionará o diretório dos fontes configurados na sua IDE.
