Durante o processo de desenvolvimento de um software por demanda, há vários participantes, responsáveis por diferentes etapas do processo. O processo utilizado pela Adapit é uma adaptação do processo Scrum, que caracteriza-se pelo desenvolvimento ágil de aplicações. O processo tem algumas adaptações para aplicar o Model Driven Development (MDD) com o WCT. O gerente de projetos desenvolve o projeto com base no guia PMBOK. A notação utilizada para especificar os requisitos do sistema é a Unified Modeling Language (UML).

 

Todo o projeto da empresa inicia com o analista de negócios. Este avalia o negócio do cliente, especifica o processo do negócio utilizando ou a UML ou o BPMN. Para determinar o preço do sistema é utilizada Análise de Ponto de Função. Em caso de acerto do sistema para com o cliente, a Adapit irá iniciar o processo de desenvolvimento. O gerente de projetos irá elaborar o escopo do projeto. O mesmo fará a comunicação com os demais perfis, dentre eles o arquiteto de sistemas, o analista de sistemas e o projetista. Para estes três perfis o WCT possibilita maximizar a produtividade, reduzindo custos operacionais da Adapit.

 

O arquiteto dos sistemas utiliza a WCT para configurar o ambiente no qual o sistema do cliente será desenvolvido. Esta configuração ocorre dentro do WCT e dita quais as tecnologias serão utilizadas para transformar um modelo de entrada em código fonte da aplicação. Assim, o arquiteto define uma máquina que irá automatizar a produção do sistema do cliente. Isto é conhecido como Model Driven Engineering (MDE) que faz uma mescla com arquiteturas para linhas de produto (Product Line Architectures PLA).

 

O analista de sistemas irá utilizar o WCT para automatizar parte do seu trabalho. Ele utiliza o Desenvolvimento Dirigido por Modelos (MDD e MDA) para transformar os requisitos do cliente em artefatos de software interpretáveis pelo WCT. Além disso, o analista utiliza o WCT para gerar modelos de software automaticamente.

 

A cada duas semanas uma versão do sistema é apresentada para o cliente. Cada versão é desenvolvida por programadores de sistemas qualificados. Tais programadores desenvolvem os recursos que não foram possíveis de serem gerados automaticamente pela WCT nas etapas de análise e projeto. Cada versão é testada iternamente e em conjunto com o cliente dos sistemas.
